# ATrust 客户端集成

本项目将 ATrust 视为可选的本机依赖，并不实现或分发其 VPN 协议、驱动、认证组件或配置。

## 集成范围

- 检测已安装的官方客户端、Windows 服务和虚拟网卡状态。
- 通过官方本机 Agent 的无凭据状态查询确认在线状态。
- 使用官方 Tray 的启动参数打开认证窗口。
- 使用官方签名的修复工具完整退出客户端；该操作可能需要管理员确认。

## 认证边界

ATrust 的认证由官方客户端完成，可能包含单点登录、验证码和多因素验证。本项目不会读取、保存、导出或重放账号、密码、Cookie、Token、验证码或会话票据。认证过期时，用户需要在官方窗口中完成验证；官方客户端自行复用其已有登录状态。

## Clash 兼容性

ATrust 使用私有控制面、虚拟网卡、DNS 和路由驱动完成隧道建立，不会产生 OpenVPN、WireGuard、Shadowsocks、VMess、Trojan 或标准 SOCKS 配置。因此本项目不提供将 ATrust 会话导出为 Clash 配置的功能。Clash 可以与已连接的 ATrust 共存，但不能替代官方客户端建立该 VPN 隧道。
