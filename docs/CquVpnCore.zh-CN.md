# CquVpnCore（C0/C1 实验阶段）

`CquVpnCore` 是 CquAutoLogin 正在构建的独立校园 VPN 核心。目标是在不依赖官方客户端、Tray、驱动、配置文件或既有终端身份的前提下，完成可验证的 VPN 连接能力。

当前仅完成 C0/C1，不应把它当作可用 VPN。

## 当前可做的事

1. 在 CquAutoLogin 的“校园 VPN”菜单中选择“连接并打开认证页”。
2. CquAutoLogin 以隐藏方式启动自身目录中的 `CquVpnCore.exe`，不启动官方客户端。
3. CquVpnCore 在默认浏览器打开认证门户；用户自行完成单点登录、多因素验证或手机验证码。
4. 可选的浏览器桥接会先发送不含页面数据的就绪回执；它只检查认证页的可见 URL、页面可见性和固定界面文字，并自动报告三态认证结果。
5. tray 显示“浏览器认证已检测到（等待后续连接能力）”，不需要用户再次点击确认。

桥接不会读取浏览器 Cookie、Local Storage、密码、Token、Authorization header、跳转参数或网络响应体。

## 当前不会做的事

- 不建立 VPN 隧道。
- 不访问控制器或校内资源。
- 不创建或复用终端身份。
- 不修改虚拟网卡、路由、DNS、防火墙或 Windows 服务。
- 不输出 Clash、OpenVPN、WireGuard 或其他通用客户端配置。
- 不代表用户处理验证码、多因素认证、CAPTCHA 或设备策略。

## 数据与进程边界

- 本地 IPC 只传递协议版本、命令、状态、操作 ID、时间戳、桥接报告类别和非敏感错误类别。
- 开发期 probe schema 只允许进程拓扑、端点类别/端口/协议/耗时、帧长度/类别、路由与 DNS 数量以及浏览器认证检测标记。
- probe 会拒绝包含密码、Token、Cookie、凭据、会话票据、授权头、载荷或命令行字段的数据。
- `CquVpnCore.exe` 监测 CquAutoLogin 的父进程；父进程退出后，core 也会退出。

## 开发门槛

进入 C2 之前，必须证明由 CquVpnCore 新生成的终端身份可以通过服务器支持的方式完成注册。进入 C3 之前，必须证明独立实现可以完成一次认证后资源协商。C4 之前不得修改系统网络配置；只有 C5 通过全新 Windows 环境、重启、重新认证和卸载官方客户端验证后，才可宣称不依赖官方客户端。

当前已有一个不接入运行时流程的本地终端身份工厂，用于生成新的、不可明文导出的 P-256 CNG 密钥；它不代表终端已注册，更不代表 VPN 已连接。

关于目前已完成的 C2 勘查、尚未获得的证据和后续验收条件，见 [C2 终端注册研究记录](./CquVpnCoreC2Research.zh-CN.md)。
